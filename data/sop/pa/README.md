# Living Spring Church — PA SOP

Standard Operating Procedures for the production / A/V (PA) team at Living Spring Church (活泉靈糧堂).

This binder is the source of truth for how we run audio, slides, and recording on Sundays. Volunteers in rotation should read everything in this folder at least once, and keep the printed booth copy of `02-pre-service.md`, `04-service-runsheet.md`, and `09-incident-playbook.md` close at hand on Sunday morning.

**Owner / approver**: Shine Zhang
**Last reviewed**: 2026-05-08

---

## How to use this SOP

- **First time on the team**: read in order README → 01 → 02 → 04 → 05 → 09 → 10. Then watch Sean's mixer walkthrough (see `training/sean-mixer-setup/notes.md`) before your first hands-on rehearsal.
- **Sunday morning reference**: `02-pre-service.md` is the checklist you actually run. Print it laminated, post in the booth.
- **When something goes wrong**: `09-incident-playbook.md` first, then ask the rotation buddy.
- **Onboarding a new volunteer**: follow `10-onboarding.md` — shadow → solo → trainer milestones.

## Conventions

- **`<!-- TBD: ... -->`** — placeholder for content that needs a specific value (gear serial number, network password, contact info). Don't ship the SOP to volunteers with TBDs visible — fill in or remove.
- **`@followup-sean`**, **`@followup-shine`** etc. — open question for that person. Search by tag.
- Each file has a footer: **Last reviewed by: <name> — YYYY-MM-DD**. Update when you make changes.
- Checklists use `- [ ]` so they can be ticked when printed or used in GitHub markdown preview.
